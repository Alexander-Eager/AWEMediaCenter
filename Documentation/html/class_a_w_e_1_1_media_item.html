bool MediaItem::addDetail ( QString  name,
QString  value 
)
virtualslot

Add a new detail with name name and value value.

If name is already a detail, then the corresponding value is set to value.

When a detail is "set", what actually happens depends on the type of the detail:

  • Array: appends the new value to the array
  • Object: does nothing, because objects are not supported.
  • Everything else: replaces the old value
Parameters
nameThe name of the detail.
valueThe value of the detail.
Returns
true if this item was changed, false otherwise.

QString MediaItem::getLocation ( ) const
virtual

Get the location for this item.

For MediaFiles, this is the path to the actual file.

For system Folders this is the path to the folder on disk.

For Folders with a media type, this is the location of the folder that contains the sub-type files (see ../type/README.md "the type README").

For MediaServices, this is either the path to the program (for JSON services) or the path to the plugin.

Returns
The location for this item.

QString MediaItem::stringFor ( const Json::Value &  val) const
private

Helper function that makes details into strings.

This is the way it is converted:

  • null becomes ""
  • strings stay what they are
  • ints become the string representation
  • bools become "Yes" or "No"
  • arrays become a comma separated list of the strings for their members
  • objects become "" (they are not supported)
Parameters
valThe value to make a string for.
